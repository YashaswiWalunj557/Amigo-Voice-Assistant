Imports:
pyttsx3: A Python library for text-to-speech conversion. This helps the assistant "speak" to the user.
speech_recognition as sr: A library for recognizing speech from the microphone. It listens to the user's commands.
wikipedia: Allows searching Wikipedia directly and retrieving summaries.
webbrowser: Lets you open web pages like YouTube, Google, etc., in the browser.
tkinter as tk: A standard Python library for creating graphical user interfaces (GUI). This builds the window where the assistant runs.
messagebox: Provides message boxes for showing simple pop-up messages.
threading: Allows the assistant to run in the background, so it doesn't block the rest of the code.
ttk: Provides themed widgets for the GUI, like better-looking buttons and labels.
cv2: OpenCV library, which is used for face detection and webcam processing.
numpy as np: Used for handling arrays, commonly used with OpenCV for image processing.
datetime: Used to get current date and time.
requests: Allows sending HTTP requests, for example, fetching weather data from a weather API.
os: For interacting with the operating system, such as opening applications or directories.
time: Provides various time-related functions, like pausing the program.
PIL.Image and ImageTk: For handling and displaying images in the GUI.
Functionality:
1. Initialize pyttsx3 Engine:
Sets up the voice engine (male or female voice) that will "speak" to the user.
2. speak() function:
This function takes a string (audio) as input and makes the assistant "speak" it using pyttsx3.
3. take_command() function:
Listens to the user’s voice through the microphone.
Converts the audio into text using Google’s speech recognition.
Returns the recognized text or "None" if it doesn’t understand.
4. Face Recognition:
Uses OpenCV to detect and recognize faces through the webcam.
It loads reference images from a folder, processes them, and compares the real-time face with those reference images.
If a face is recognized, the assistant proceeds to speak and interact with the user.
If no face is recognized, it continues to capture video frames.
5. Weather:
The get_weather() function takes a city name and fetches weather data from the OpenWeather API.
It returns temperature, humidity, pressure, and description of the weather.
6. mic_animation():
A function to animate the microphone icon while the assistant listens for commands. It changes the mic image to give a visual cue that the assistant is listening.
7. start_assistant():
Starts the assistant in a separate thread to avoid blocking the rest of the program.
8. Assistant Logic (run_assistant() function):
This is the main function that runs in a loop, continuously listening for commands.
If a recognized command is heard (like "who is", "weather", "date", etc.), it performs the relevant action:
Retrieves information from Wikipedia.
Fetches weather data.
Tells the current date or time.
Opens specific websites like Google or YouTube.
Stops when the command "sleep" is heard.
9. GUI Design:
A window (root) is created using tkinter, which includes:
A title with the label "Amigo Assistant".
A text box to display the assistant’s responses.
A microphone button to activate the assistant.
Stop and clear buttons for controlling the assistant.
10. update_text():
This function updates the text area in the GUI with the assistant’s responses.
11. clear_text():
Clears all the content from the text box in the GUI.
12. mic_button_pulse():
Creates a "pulse" animation effect for the microphone button (grows and shrinks the button size to create a pulsing effect).
13. start_assistant_with_pulse():
Starts the assistant with the animation effect.
14. Button and Animation:
Various UI elements (buttons for stopping, clearing output) are created and animated for a better user experience.
The buttons change their appearance slightly when hovered over or clicked (raised to flat).
15. GUI Loop:
The root.mainloop() command keeps the GUI window running, awaiting user interaction.
How It All Works:
When you press the microphone button, the assistant starts listening for your voice command.
Depending on what you say (like "What is the time?", "Open YouTube", or "Who is Albert Einstein?"), it will either fetch the time, open websites, or look up information on Wikipedia.
It responds by speaking the information out loud and also displays the response in the text box.
If the user says "sleep," the assistant stops.
Final Flow:
Assistant Initialization: The assistant is started and listens for commands.
Face Recognition: It ensures that the user is recognized before giving control to the assistant.
Voice Commands: The user speaks, and the assistant responds with actions or information.
GUI: The assistant's interface provides a clean and interactive way to communicate with the user.
